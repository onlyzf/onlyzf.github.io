当我们讨论游戏程序的时候我们在讨论什么？

  • 游戏应用——Game Application，手机上直接运行的游戏，比如：Invasion、Brutal Age
  • 人——Programmer，把游戏制作人策划的想法和各种资源整合起来，实现成能游玩的游戏的人

我们现在主要讨论的是人，或者说是游戏程序这个行业

游戏程序的历史

话说自上古时代 Computer 发明以来，越来越多的工作需要用 Computer 才能完成。有那么一群想在上班时间摸鱼的人对不能用 Computer 摸鱼表达强烈的不满，甚至扬言如果 Computer 不能摸鱼还要它何用，不如卖了换钱。有一群善良的 Programmers 听到了上帝的召唤：“要能摸鱼！”，于是就有了世界上第一批 Game Programmers。

以上纯属胡诌，娱乐是人吃饱后的天性，就算把人扔到空无一物的监狱里也能想出摸鱼的方法来。我只想说明一点，我国社会主要矛盾已经转化为人民日益增长的美好生活需要和不平衡不充分的发展之间的矛盾。

从游戏程序的诞生就可以看出，策划、美术、音乐、运营等等这些岗位都是游戏开发发展到很成熟的阶段衍生出来的岗位，最早从事策划美术音乐项管的统统是程序员。因为早期还没有游戏引擎甚至没有显卡的年代，只有程序员才懂得如何在那些该死的屏幕上通过拼像素点儿显示出想要的图形。像这样：

典型的游戏机的历史能在网上找到一大堆，我就不班门弄斧了。重点回忆下我的游戏及程序的经历。 1991、1992 年，放学去最早的游戏机房玩 FC(NES) 上的魂斗罗、双截龙 2，没钱的时候就看别人玩。 FC

1993 年，有了我第一台个人电脑，CPU 8086，640KB 内存，无硬盘，3.5 寸软驱，用电视机当显示器。操作系统是 MSDOS 3.3。那几年我开始进入程序员的大门，自学了 GWBasic，最好的成绩就是照着书用字符模式做了个打飞机。比下面这个图示还 Low。

8086 CPU 基本上是最老的一代 PC CPU，只支持 1MB( 1024MB = 1GB，现在手机的内存都 4GB 起了) 内存。当时在 DOS 下面操作系统启动完成后只剩 600KB 左右内存，居然还有波斯王子可以玩，可以想像老程序员能把硬件性能榨到极致。 嗯，那会儿大部分游戏的分辨率是 320x240。

1996 年，3dfx 发布了 Voodoo 3D 显卡，这也是我第一次听说 3D 这个词。那时候做梦都想试试真正的 3D 游戏。 在没有 3D 显卡之前，所有图形的绘制都是由程序员直接调用汇编接口刷到显示器上的 没用 OpenGl 硬件加速渲染的《Quake2》果 使用 3dfx 卡渲染后的《Quake2》果

中午去街机厅玩 3D 格斗游戏，铁拳 2，应该是最早的 3D 游戏

后高中时代主要是在 PS 上玩红警，这也是我玩的第一个联机游戏。在游戏机房租两台主机，通过对战线直连。这其中的通信、同步等开发技术直到我做格斗游戏的时候才真正理解透。

大学期间玩的最多的是星际争霸、帝国时代、还有用 WinKawaks 玩 NeoGeo 街机版的拳皇 98。是的，虚拟机这类技术早就在游戏领域应用甚广了，它的复杂之处其实是性能优化。当 PC 的性能远远超过几年前的街机的时候，做个模拟器也是轻而易举的事。

毕业后终于有机会进入游戏行业，第一次跟网游最近的工作就是脱机外挂，这个脱机应该是我做过甚至见过的最复杂的脱机。它的 NB 之处在于我们用 C++ Builder 重新实现了一遍游戏客户端的主逻辑，并且为了自动挂机，把原游戏里的 3D 地图全部载入进来，并在上面手工生成导航网格，地图内寻路使用 A*，地图间寻路使用 Dijkstra 算法。挂机 AI 使用基于优先级的有限状态机。最终的效果是用户可以在脱机工具上全自动打怪升级跑商攒钱，也可以手动移动商店购买等等操作。

一年后有幸加入盛大，跟随了功夫小子这款游戏上线前到上线很长一段时间。当时的开发是不分客户端服务器的，主语言是 C++，我加入后把 Lua 集成到客户端作为 AI 脚本给功夫小子添加了不少 PVE 的内容。客户端使用的渲染引擎是当时非常著名的 renderware，这是个很奇特的引擎，是我见过的唯一的非面向对象的引擎。它的接口调用方法和直接调用 DX 或 OpenGL 非常想，现在回想起来似乎只是比 DX 更高层次的封装。在这个游戏整个运营期间还了解了

再过了几年，我们功夫小子的核心成员出来创业成立了自己的公司。还是做网络格斗游戏。客户端使用 Ogre 作为 3D 渲染引擎，CEGUI 做 GUI 交互，Lua 作为客户端业务逻辑脚本；服务器还是使用 C++，这次把 Lua 也嵌入了服务器，和客户端同架构、代码公用。程序员除了有专门的 3D 引擎工程师外其他人还是不分，一个大的功能模块拉通了一起开发。我们做了大量的编辑器，场景编辑器、特效编辑器、AI 编辑器、UI 编辑器、动作编辑器等等，客户端尽量做到所见即所得，编辑器导出的资源可以直接由美术策划在游戏中查看，大大提升开发效率。
